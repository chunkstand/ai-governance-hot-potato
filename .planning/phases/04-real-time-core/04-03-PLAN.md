<objective>
Implement server-authoritative game state broadcast, heartbeat/ping-pong health monitoring, and exponential backoff reconnection. This plan creates the game state manager that broadcasts full state to spectators, adds 30-second heartbeat to detect stale connections, and implements resilient reconnection with exponential backoff (max 10 retries) and 1-3 second initial delay per user decision.

Purpose: Without server-authoritative state, clients desynchronize. Without heartbeat, connections leak. Without proper reconnection, brief network issues break the spectator experience. This delivers the resilience layer that makes the real-time system production-ready.
Output: Full game state broadcast to all spectators, heartbeat monitoring active, reconnection logic with visual feedback.
</objective>

  <action>
    Build heartbeat/ping-pong for connection health monitoring:
    
    1. Create backend/src/socket/heartbeat.ts:
       - startHeartbeat(io) - sets up ping-pong on all namespaces
       - For each socket connection:
         * Set socket.data.lastPing = Date.now() on connect
         * Set 30-second interval to emit 'ping' to all sockets
         * Listen for 'pong' response and update lastPing
         * Check for stale connections: if lastPing > 60 seconds ago, force disconnect
       - isAlive(socket) - returns true if lastPing within 60 seconds
       - stopHeartbeat() - clears interval (for graceful shutdown)
       - Track heartbeat metrics: active connections, stale disconnections
    
    2. Update namespaces to use heartbeat:
       - In spectator.ts and game.ts: call heartbeat.registerSocket(socket) on connect
       - On 'disconnect' reason: log if heartbeat timeout vs client disconnect
    
    3. Client-side heartbeat response (update frontend/js/socket-client.js):
       - On 'ping' event from server: emit 'pong' response immediately
       - Track last server ping time for client-side health monitoring
    
    This implements RTC-05 (30-second heartbeat) with 60-second stale detection per success criteria.
    Research notes from PITFALLS.md: Without heartbeat/ping-pong, memory grows until OOM kills.
